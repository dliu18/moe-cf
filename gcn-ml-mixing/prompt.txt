I would now like to do data mixing on the ml-1m dataset while training LightGCN.

This is the data mixing setup: 
* we have group labels for all n users. the users are divided into g groups:
	* n_g is the number of users in group g
* Let bpr(g) denote the bpr loss value summed over only users in group g. So the current (vanilla BPR) loss would be: l = \sum_g bpr(g)
* For data mixing let s be the source group and all other groups are augmentation groups. 
* Let n_aug be the number of users in the augmentation groups. 
* Define data mixing parameters alpha_aug and alpha_mix
	* alpha_aug is a scalar that scales the significance of the augmentation dataset 
	* alpha_mix is a vector of cardinality g-1. alpha_mix assigns a ratio to each augmentation group where the sum of the ratios is 1.
* Given group labels, a source group label, and alpha_aug and alpha_mix, data mixing makes the following modifications:
	* training: 
		* the loss function is: l_mix = bpr(s) + n_aug * alpha_aug * (\sum_{g\s} (\alpha_g / n_g) bpr(g))
		* if alpha_aug = 1 and alpha_g = n_g / n_aug then l_mix is the original bpr loss
	* testing:
		* ranking metrics are evaluated on _only_ users that belong in the source augmentation group.
