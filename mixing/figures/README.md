# Figures

This directory contains scripts and outputs for publication figures.

## Overall Figures

The "overall" figure is generated by:

```bash
python mixing/figures/generate_overall_grouped_bars.py --recdim <recdim> --metric <metric_name>
```

Where `<metric_name>` is one of:
- `precision`
- `recall`
- `ndcg`

`recdim` is provided via CLI, and model/data combinations are inferred from the CSV folder structure.
The script also supports:

```bash
python mixing/figures/generate_overall_grouped_bars.py --recdim <recdim> --metric <metric_name> --debug
```

`--debug` prints loaded-file and per-panel data diagnostics.

### End-To-End Pipeline

1. Run the notebook cell that exports grouped-bar plotting CSVs:
   - Notebook: `mixing/analyze_mixing_validation.ipynb`
   - Cell label: `# Figure: grouped bars by source label, with baselines + top mix + average mix (test-set metrics)`
2. That cell writes plotting data to:
   - `mixing/figures/plotting_data/overall/{dataset}/{model}/recdim_{RECDIM}/`
   - Files include:
     - `precision__grouped_bars_test.csv`
     - `recall__grouped_bars_test.csv`
     - `ndcg__grouped_bars_test.csv`
3. Run the overall figure script:
   - `python mixing/figures/generate_overall_grouped_bars.py --recdim 64 --metric ndcg`
4. Collect final figure PDFs from:
   - `mixing/figures/pdfs/overall/`
   - Filename pattern:
     - `overall_{metric}_recdim_{RECDIM}.pdf`

### Data/Model Mapping Used In Figure

- Datasets:
  - `ml-1m` -> `MovieLens-1M`
  - `lastfm-asia` -> `LastFM-Asia`
- Models:
  - `mf` -> `Matrix Factorization`
  - `lgn` -> `LightGCN`

### Figure Layout and Style (Overall Script)

- Subplot columns: datasets (`MovieLens-1M`, `LastFM-Asia`)
- Subplot rows: models (`Matrix Factorization`, `LightGCN`)
- Y-axes are not shared across subplots.
- Subplot title format: `{Dataset Name} {Model Name}`
- X-axis: `Source Group`
- Y-axis: selected metric (title case)
- `NDCG` is displayed in all caps.
- Bars shown:
  - `Proportional` (black)
  - `No Augmentation` (`#808080`)
  - `Stratified` (`#7570b3`)
  - `Data Mix` (`#d95f02`)
- `Avg Mix` is not plotted.
- Only `Data Mix` bars receive `% change vs proportional` annotation.
- Asterisks (`*`) are added to comparator bars when the comparator-vs-proportional difference is statistically significant (see significance testing below).
- Source-group ticks are dataset-specific for each subplot.
- Column widths are proportional to each dataset's number of source groups.
- A single figure-level legend is used.
- No figure super-title is used.

### Statistical Significance (Overall Figure)

For each `(dataset, model, recdim, source_group, bar_key, metric)`:

1. Pull trial-level metric values from raw test-eval CSVs for:
   - proportional
   - comparator (`no_augmentation`, `stratified`, or `top_1`)
2. Run a nonparametric bootstrap on the difference in means:
   - resample each group with replacement
   - compute `mean(comparator) - mean(proportional)` per iteration
   - default iterations: `3000`
3. Compute a two-sided 95% percentile confidence interval (`alpha=0.05`).
4. Mark significance (`*`) if CI excludes `0`:
   - lower bound `> 0`, or upper bound `< 0`.

If either group has fewer than 2 samples, no significance marker is added.

### Expected Inputs and Checks

The script expects all required CSVs to exist for:
- datasets: `ml-1m`, `lastfm-asia`
- models: `mf`, `lgn`
- selected `recdim`
- selected metric CSV

If any file is missing, the script raises a clear `FileNotFoundError` with the expected path.

### Troubleshooting

- If output CSV directories are missing, rerun the grouped-bar export cell in the notebook.
- If `ModuleNotFoundError` appears (for example `pandas`), install required Python packages in your environment.
- If metric name is rejected, use exactly one of: `precision`, `recall`, `ndcg`.

## Prop Vs. Stratified Row Figure

This figure is generated by:

```bash
python mixing/figures/generate_prop_vs_stratified_row.py --model <lgn|mf> --metric <metric_name>
```

Two-column layout option:

```bash
python mixing/figures/generate_prop_vs_stratified_row.py --model <lgn|mf> --metric <metric_name> --two-column
```

Optional confidence bands:

```bash
python mixing/figures/generate_prop_vs_stratified_row.py \
  --model lgn \
  --metric ndcg \
  --ci-bands \
  --bootstrap-iters 5000 \
  --ci-level 95 \
  --seed 42
```

### End-To-End Pipeline

1. Ensure experiment CSVs exist in:
   - `mixing/experiments/prop_vs_stratified/results/`
2. Define subplot dataset/source-group pairs in:
   - `mixing/figures/prop_vs_stratified_pairs.json`
   - Format:
     - `{"pairs": [{"dataset": "...", "source_group": "..."}, ...]}`
3. Run figure generation:
   - `python mixing/figures/generate_prop_vs_stratified_row.py --model lgn --metric ndcg`
4. Collect output PDF from:
   - `mixing/figures/pdfs/prop_vs_stratified/`
   - Filename pattern:
     - `prop_vs_stratified__{model}__{metric}.pdf`
     - `prop_vs_stratified__{model}__{metric}__two_column.pdf` (when `--two-column` is enabled)

### Inputs

- Script:
  - `mixing/figures/generate_prop_vs_stratified_row.py`
- Pair config:
  - `mixing/figures/prop_vs_stratified_pairs.json`
- Results data:
  - `mixing/experiments/prop_vs_stratified/results/*.csv`

### Plot Definition

- One row of subplots, one subplot per config pair (`dataset`, `source_group`).
- With `--two-column`: one subplot row per dataset (pairs grouped by dataset).
- X-axis: embedding dimension (`recdim`), numeric and log-scaled (base 2).
- Y-axis: relative percent change of stratified vs proportional:
  - `100 * (mean(stratified) - mean(proportional)) / mean(proportional)`
- Line style:
  - Black line
  - Point marker at each recdim
- Horizontal baseline at 0%.

### Titles and Labels

- Subplot title:
  - `{dataset name}`
  - `Source Group: {group id}` on the next line
- Y-axis label:
  - `{metric name} Δ% Vs. Proportional`
- Dataset display mapping:
  - `ml-1m` -> `MovieLens-1M`
  - `lastfm-asia` -> `LastFM-Asia`
- Metric display:
  - `NDCG` is shown in all caps.

### Metric Selection

- `--metric` accepts either:
  - Base metric name: `precision`, `recall`, `ndcg`
  - Full metric column: e.g., `recall@20`, `ndcg@100`
- If a base name is given, dataset-specific default `k` is used when available:
  - `ml-1m`: `@20`
  - `lastfm-asia`: `@100`

### Model Handling

- `--model` is required (`lgn` or `mf`).
- The script filters rows to this model before aggregation/comparison.

### Confidence-Interval Bands

- Enable with `--ci-bands`.
- Bands are light gray shaded regions around the % change line.
- CI method:
  - Nonparametric bootstrap over trial-level rows within each recdim and baseline.
  - Resample proportional and stratified trials with replacement.
  - Recompute `% change` per bootstrap draw.
  - Use percentile CI at `--ci-level` (default 95).
- Controls:
  - `--bootstrap-iters` (default `5000`)
  - `--ci-level` (default `95`)
  - `--seed` (default `42`)

### Typography

This figure uses the same font family/sizing style as the overall figure script.

## Similarity Vs. Top-Mix Figure

This figure is generated by:

```bash
python mixing/figures/generate_similarity_vs_topmix.py \
  --model <lgn|mf> \
  --recdim <recdim>
```

Example:

```bash
python mixing/figures/generate_similarity_vs_topmix.py \
  --model lgn \
  --recdim 64 \
  --debug
```

Recorded run:

```bash
python mixing/figures/generate_similarity_vs_topmix.py --model lgn --recdim 4 --top-k 1 --debug
```

### End-To-End Pipeline

1. Ensure preprocessed LightGCN split outputs exist for each dataset:
   - `LightGCN/data/{dataset}/train_full.txt`
   - `LightGCN/data/{dataset}/test.txt`
   - `LightGCN/data/{dataset}/user_labels.pkl`
2. Ensure test-eval CSVs exist:
   - `mixing/test/{dataset}/{model}/{recdim}/test_eval__feature-*__source-*.csv`
3. Ensure subplot pairs config exists:
   - `mixing/figures/prop_vs_stratified_pairs.json`
4. Run:
   - `python mixing/figures/generate_similarity_vs_topmix.py --model lgn --recdim 64`
5. Collect output PDFs from:
   - `mixing/figures/pdfs/similarity_vs_topmix/`

### Inputs

- Script:
  - `mixing/figures/generate_similarity_vs_topmix.py`
- Pairs config:
  - `mixing/figures/prop_vs_stratified_pairs.json`
- Splits root:
  - `LightGCN/data/`
- Test-eval root:
  - `mixing/test/`

### Similarity Definition

For source group `i` and augmentation group `j`:

- User-item set for a user = union of train and test interactions:
  - `I_u = train_full(u) ∪ test(u)`
- User-level similarity:
  - `J(u,v) = |I_u ∩ I_v| / |I_u ∪ I_v|`
- Group-level similarity:
  - average of `J(u,v)` over all user pairs where `u ∈ group i` and `v ∈ group j`
- Only cross-group pairs are used in this workflow (`i != j` in configured usage).

Current implementation for plotting uses **normalized discrepancy** (not Jaccard) on item subspaces:

- Fixed rank `d = 4`
- Projection definition: `P_g = V_g V_g^T`
- X-axis discrepancy:
  - `dist(i, j) = ||P_i - P_j||_F^2 / ||P_i||_F^2`

### Top-Mix Alpha Component

- For each `(dataset, source_group)` panel, the script reads the corresponding test-eval file.
- It uses `top_1..top_k` rows (controlled by `--top-k`) and averages alpha components per augmentation group.
- For `--top-k 1`, behavior is the same as using `top_1` only.

### Produced Figures

Two row-layout figures are produced (one subplot per configured pair):

1. Grouped bars:
   - x-axis: augmentation group
   - bar A: normalized discrepancy(target, augmentation)
   - bar B: average top-k mix component for that augmentation group
2. Scatter:
   - x-axis: normalized discrepancy with target group
   - y-axis: mixing ratio
   - one point per augmentation group

Both figures now include an additional **rightmost bar-chart subplot**:

- x-axis: target group label
- y-axis: `Augmentation Ratio`
- value = average `alpha_aug` over `top_1..top_k` rows for each target group

Subplot title format:
- `{dataset name}`
- `Target Group: {group id}` on the next line

### Output Files

- Grouped bars:
  - `mixing/figures/pdfs/similarity_vs_topmix/similarity_vs_topmix_grouped_bars__{model}__recdim_{recdim}__topk_{k}.pdf`
- Scatter:
  - `mixing/figures/pdfs/similarity_vs_topmix/similarity_vs_topmix_scatter__{model}__recdim_{recdim}__topk_{k}.pdf`

### Progress + Pair Counts

- Before similarity computation, the script prints:
  - number of users in source group
  - number of users in each augmentation group
  - total user-pair count per `(source_group, augmentation_group)`

### Similarity Cache (Reuse Across Runs)

- Default cache path:
  - `mixing/figures/similarity_cache.json`
- Override via:
  - `--similarity-cache <path>`
- Cache key is symmetric by design:
  - `sim(i,j)` and `sim(j,i)` share the same stored key
  - avoids redundant recomputation
- Cache is loaded at start and saved at end of each run.

### Additional CLI

- `--config` to override pairs JSON path.
- `--splits-root` to override split-data root.
- `--test-root` to override test-eval root.
- `--out-dir` to override output directory.
- `--top-k` to average alpha components across `top_1..top_k`.
- `--debug` to print cache hits/misses and additional diagnostics.

## Intergroup Similarity Vs. Dimension Figure

This figure is generated by:

```bash
python mixing/figures/generate_similarity_vs_dim.py --debug
```

Optional dataset selection:

```bash
python mixing/figures/generate_similarity_vs_dim.py --datasets ml-1m,lastfm-asia --debug
```

Two-column layout option:

```bash
python mixing/figures/generate_similarity_vs_dim.py --datasets ml-1m,lastfm-asia --two-column --debug
```

### End-To-End Pipeline

1. Ensure LightGCN split outputs exist for each dataset:
   - `LightGCN/data/{dataset}/train_full.txt`
   - `LightGCN/data/{dataset}/test.txt`
   - `LightGCN/data/{dataset}/user_labels.pkl`
2. Run:
   - `python mixing/figures/generate_similarity_vs_dim.py --debug`
3. Collect outputs from:
   - `mixing/figures/pdfs/similarity_vs_dim/`

### Inputs

- Script:
  - `mixing/figures/generate_similarity_vs_dim.py`
- Data roots:
  - `LightGCN/data/`

### Core Definitions

For each dataset and group `i`:

1. Build user-item interactions from union of train/test:
   - user item set = `train_full(u) ∪ test(u)`
2. Construct group adjacency matrix `A_i` by row-filtering users in group `i`.
3. Compute rank-256 SVD once per group:
   - `A_i ≈ U_i Σ_i V_i^T`
4. For each embedding dimension `d` in `2..256`, truncate SVD factors.

Distance for source group `i` vs augmentation group `j`:

- `dist(i, j) = ||P_i - P_j||_F^2 / ||P_i||_F^2`
- Asymmetric because normalization is by `||P_i||_F^2`.

### Projection Variants (Both Output)

The script outputs both projection choices:

1. `VΣV^T` (weighted projection):
   - `P_i = V_i Σ_i V_i^T`
   - output suffix: `__vsvt`
2. `VV^T` (subspace projection):
   - `P_i = V_i V_i^T`
   - output suffix: `__vvt`

### Main Figure Outputs (Per Dataset)

For each dataset and each projection variant:

- Subplot row: one row
- Subplot columns: all source groups in that dataset
- For each source group subplot:
  - x-axis: embedding dimension (`2..256`, log scale base 2)
  - y-axis: normalized group distance
  - one curve per augmentation group
  - title: `Source Group: {group}`
- Shared legend at top
- Group colors are consistent across all subplots within a dataset

Files:

- `intergroup_similarity_vs_dim__ml-1m__vsvt.pdf`
- `intergroup_similarity_vs_dim__ml-1m__vvt.pdf`
- `intergroup_similarity_vs_dim__lastfm-asia__vsvt.pdf`
- `intergroup_similarity_vs_dim__lastfm-asia__vvt.pdf`
- With `--two-column`, filenames include `__two_column` before the projection suffix:
  - `intergroup_similarity_vs_dim__{dataset}__two_column__vsvt.pdf`
  - `intergroup_similarity_vs_dim__{dataset}__two_column__vvt.pdf`

### Additional Variance Outputs

The script also outputs variance summaries (one file per projection variant):

- `intergroup_distance_variance_vs_dim__vsvt.pdf`
- `intergroup_distance_variance_vs_dim__vvt.pdf`
- With `--two-column`:
  - `intergroup_distance_variance_vs_dim__two_column__vsvt.pdf`
  - `intergroup_distance_variance_vs_dim__two_column__vvt.pdf`

Figure layout:

- Subplots side-by-side by dataset
- x-axis: embedding dimension (`2..256`, log scale)
- y-axis: `Variance of Group Distances`
- one curve per source group
- subplot title: dataset name

### Additional CLI

- `--datasets` comma-separated datasets to run (default: `ml-1m,lastfm-asia`)
- `--splits-root` to override `LightGCN/data` root
- `--out-dir` to override output directory
- `--two-column` stacks subplot panels vertically and appends `__two_column` in output filenames
- `--debug` for progress diagnostics
